Desenvolvendo um sistema WEB a partir da Análise da dados

Desenvolver um sistema WEB compreende informatizar tanto o front-end quanto o back-end.
No front-end iremos trabalhar as páginas que serão mostradas ao usuário.
No back-end iremos trabalhar como os dados, passados pelo usuário ou criados pelo sistema, serão armazenados
em um banco de dados.

Existem várias metodologias para se desenvolver um sistema WEB, mas basicamente se resume em:
1º Estabelecer as linguagens de programação a serem utilizadas. Exemplo:
    - No front-end: HTML, CSS e JavaScript
    - No back-end: mysqli e PHP
2º Definir a sistemática de padronização, documentação e estrutura, tipo:
    - Padronização:
        => Definir a convençao de nomenclatura para pastas, arquivos, variáveis, etc
        => Definir o layout a ser utilizado
        => Definir a paleta de cores, etc.
    - Documentação:
        => Guardar todos os documentos da análise de dados
        => Sistematizar, na escrita do código de programação, a forma de comentários
        => Criar arquivos de cópias de segurança, etc.
    - Estrutura:
        => Criar o esquema de pasta e subpastas para o projeto a ser desenvolvido
        => Definir a estrura de arquivos a serem desenvolvidos
3º Desenvolver o projeto:
    - Seguir os objetivos traçados na análise e levantamento de requisitos
    - Seguir as diretrizes de padronização, documentação e estrutura
4º Efetuar testes e otimizar o desempenho do sistema
5º Implantar o site na internet e o sistema de manutenção

### Estrutura de Arquivos
1. **index.php** - Página inicial.
2. **db.php** - Conexão com o banco de dados.
3. **matrizes.php** - CRUD para matrizes.
4. **partos.php** - CRUD para partos.
5. **crias.php** - CRUD para crias.
6. **vacinas.php** - CRUD para vacinas.
7. **vacinas_matrizes.php** - CRUD para vacinas de matrizes.
8. **vacinas_crias.php** - CRUD para vacinas de crias.
9. **procedimentos.php** - CRUD para procedimentos.
10. **procedimentos_crias.php** - CRUD para procedimentos de crias.
11. **alimentacao_matrizes.php** - CRUD para alimentação de matrizes.
12. **pesagem_matrizes.php** - CRUD para pesagem de matrizes.
13. **coberturas.php** - CRUD para coberturas.

### 1. Conexão com o Banco de Dados (db.php)
```php
<?php
$servername = "localhost";
$username = "root";
$password = "";
$dbname = "fazendaEtec";

// Criar conexão
$conn = new mysqli($servername, $username, $password, $dbname);

// Verificar conexão
if ($conn->connect_error) {
    die("Conexão falhou: " . $conn->connect_error);
}
?>
```

### 14. Editar e Excluir para as Tabelas Restantes
Para as operações de edição e exclusão, você pode seguir a mesma estrutura que usamos anteriormente. Aqui está um exemplo para a tabela `pesagem_matrizes`:

#### c. Editar Pesagem para Matriz (edit_pesagem_matriz.php)
```php
<?php
include 'db.php';

$id = $_GET['id'];
$sql = "SELECT * FROM pesagem_matrizes WHERE id=$id";
$result = $conn->query($sql);
$row = $result->fetch_assoc();

if ($_SERVER['REQUEST_METHOD'] == 'POST') {
    $matriz_id = $_POST['matriz_id'];
    $peso = $_POST['peso'];
    $data = $_POST['data'];

    $sql = "UPDATE pesagem_matrizes SET matriz_id='$matriz_id', peso='$peso', data='$data' WHERE id=$id";
    if ($conn->query($sql) === TRUE) {
        header('Location: pesagem_matrizes.php');
    } else {
        echo "Erro: " . $sql . "<br>" . $conn->error;
    }
}
?>
<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <title>Editar Pesagem para Matriz</title>
    <link rel="stylesheet" href="style.css">

    <h1>Editar Pesagem para Matriz</h1>
    <form method="post">

</html>
```

#### d. Excluir Pesagem para Matriz (delete_pesagem_matriz.php)
